Plone Mockup

Plone Mockup is an attempt to separate Plone, the user interface, from Plone, the server. It is realised with Patterns (http://patternslib.com), a library of rich interaction patterns. Patterns are triggered by CSS classes and configured via HTML5-data attributes.

The mockup is a fully functional prototype of Plone and its bundles. As it is realised entirely in HTML+JS, it can be run in a browser without the need for a full Plone installation. Therefore making it easy to showcase Plone's UI or developing a new theme. Using diazo/deliverance it is possible to integrate other backends alongside or instead of Plone.

Bundles

Diazo

Diazo toolbar exposes the diazo rules applied to a theme (or not) in order to map it to the backend. This helps when writing and debugging diazo rules. iframe.js is the script used to create the diazo debugging toolbar.

To build and use the diazo toolbar bundle, follow the steps below:

$ make diazo
$ cp build/diazo* path/to/diazo/lib/diazo/debug_resources
Demo (not working yet)

Widgets

plone.app.widgets are an alternative to the current Plone widgets, with enhanced features and modern design.

To build and use the widgets bundle, follow the steps below:

$ make widgets
$ cp build/widgets* path/to/plone.app.widgets/plone/app/widgets/static
Demo

Toolbar

plone.app.toolbar separates the content management interface elements from the design, placing all functionality neatly into a toolbar at the top. This enormously simplifies theming and also usability for content managers.

To build and use the toolbar bundle, follow the steps below:

$ make toolbar
$ cp build/toolbar* path/to/plone.app.toolbar/plone/app/toolbar/static
Demo (not working yet)

Standard Tiles

plone.app.standardtiles are reusable visual elements outside content area, eg. logo, footer, ... This will eventually deprecate viewlets and portlets (forever!)

To build and use the standardtiles bundle, follow the steps below:

$ make standardtiles
$ cp build/standardtiles* path/to/plone.app.standardtiles/plone/app/standardtiles/static
Demo (not working yet)

Deco

plone.app.deco is the new layout editor for the content area. Provides user friendly way (drag-n-drop) of adjusting the layout of the page.

To build and use the deco bundle, follow the steps below:

$ make deco
$ cp build/deco* path/to/plone.app.deco/plone/app/deco/static
Demo (not working yet)

Use it now

git clone git://github.com/plone/mockup.git
cd mockup
make bootstrap

Until Patterns is under heavy development.

cd mockup
mkdir ext
cd ext
git clone git://github.com/Patternslib/Patterns.git
cd ..
./node_modules/jamjs/bin/jam.js link ext/Patterns

To build all bundles:

make

Run tests:

make test
