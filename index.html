Plone Mockup

Plone Mockup is an attempt to separate Plone, the user interface, from Plone, the server. It is realised with Patterns (http://patternslib.com), a library of rich interaction patterns. Patterns are triggered by CSS classes and configured via HTML5-data attributes.

The mockup is a fully functional prototype of Plone. As it is realised entirely in HTML+JS, it can be run in a browser without the need for a full Plone installation. Therefore making it easy to showcase Plone's UI or developing a new theme. Using diazo/deliverance it is possible to integrate other backends alongside or instead of Plone.

Quickstart

Install

git clone git://github.com/plone/mockup.git
cd mockup
make bootstrap

Until Patterns merges our pull requests we also need link to development version of Patterns library.

cd mockup
mkdir ext
cd ext
git clone git://github.com/garbas/Patterns.git
cd ..
./node_modules/jamjs/bin/jam.js link ext/Patterns

Build

To build all type:

make

To build for specific project type:

make <project>

Where project can be any existing intagrations

Test

make test
